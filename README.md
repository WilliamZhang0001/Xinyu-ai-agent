# 心语| AI 超级智能体项目

## 项目介绍


通过开发 **AI 恋爱大师应用 + 拥有自主规划能力的超级智能体**，掌握新时代必知必会的 AI 核心概念、AI 实用工具、AI 编程技术、AI 框架原理、AI 调优技巧.


`AI 恋爱大师应用` 可以依赖 AI 大模型解决用户的情感问题，支持多轮对话、基于自定义知识库进行问答、自主调用工具和 MCP 服务完成任务，比如调用地图服务获取附近地点并制定约会计划。

## 为什么要带做这个项目？

在项目前期的用户调研中，我访谈了身边多位同学、朋友，以及详尽的市场调研。大家在“寻求情感建议”这件事上，都提到了一个共同痛点——不是没人倾诉，而是不知道如何获得“可信、可执行、可持续跟进”的恋爱建议。
在实际需求里，他们普遍卡在几个核心环节：

1）网上答案碎片化、模板化，没有针对性
- 小红书/知乎的情感分析太宏观，不贴合个人具体场景。
- 大多数“万能话术”只解决表面，不适合真实关系动态。

2）朋友能倾诉，但难提供专业建议
- 情绪支持可以给，但实质性的行动建议常常不够专业。
- 很多人反映“朋友一句话让我更焦虑”。

3）传统情感咨询成本高、门槛高
- 专业顾问收费高，且无法随时响应。
- 很多人不愿将私密情感问题交给真人咨询师。

4）真正困难的不是“知道怎么回消息”，而是“不知道怎么规划整个关系的发展路径”
- 想约会但不知道去哪儿、怎么计划？
- 想复合但不清楚先建立什么信号？
- 想表达需求但不知道怎么措辞才能让彼此舒服？

所以，我们想做的心语核心价值并不是“代写情话”，而是AI 驱动的情感决策辅助系统。
我们不是仅回答“怎么回消息”，而是让 AI 成为一个持续陪伴式的恋爱顾问，覆盖：

1）理解真实用户背景（多轮对话 + 持久记忆）
AI 能记住用户与对象的关系、相处节奏、争吵历史，提供更贴合实际的建议。

2）结合知识库给专业建议（RAG）
我们构建了基于心理学、沟通学、关系维护理论的知识库，AI 的判断不是“拍脑袋”，而是基于专业内容支撑。

3）通过工具与 MCP 服务给出“可执行的行动方案”
例如：
- 通过地图服务自动查找用户附近的高评分约会地点
- 利用 ReAct Agent 自动规划一个完整约会流程
- 自动生成 PDF 约会计划书
- 调用搜索与资源下载工具寻找礼物、话题、节日活动方案
这是从“回答问题”升级到“帮你把事做完”。

选题新颖、业务真实，区别于增删改查的 “烂大街” 项目，通过实战大量新技术和企业级别应用场景，将必知必会的 **AI 技术一网打尽**，成为 AI 时代企业的香饽饽，提升简历和求职竞争力。

## 项目功能梳理

项目中，我们开发了一个 AI 恋爱大师应用、一个拥有自主规划能力的超级智能体，以及一系列工具和 MCP 服务。

具体需求如下：

- AI 恋爱大师应用：用户在恋爱过程中难免遇到各种难题，让 AI 为用户提供贴心情感指导。支持多轮对话、对话记忆持久化、RAG 知识库检索、工具调用、MCP 服务调用。
- AI 超级智能体：可以根据用户的需求，自主推理和行动，直到完成目标。
- 提供给 AI 的工具：包括联网搜索、文件操作、网页抓取、资源下载、终端操作、PDF 生成。
- AI MCP 服务：可以从特定网站搜索图片。



## 用哪些技术？

项目以 Spring AI 开发框架实战为核心，涉及到多种主流 AI 客户端和工具库的运用。

- Java 21 + Spring Boot 3 框架
- ⭐️ Spring AI + LangChain4j
- ⭐️ RAG 知识库
- ⭐️ PGvector 向量数据库
- ⭐ Tool Calling 工具调用 
- ⭐️ MCP 模型上下文协议
- ⭐️ ReAct Agent 智能体构建
- ⭐️ Serverless 计算服务
- ⭐️ AI 大模型开发平台百炼
- ⭐️ Cursor AI 代码生成
- ⭐️ SSE 异步推送
- 第三方接口：如 SearchAPI / Pexels API
- Ollama 大模型部署
- 工具库如：Kryo 高性能序列化 + Jsoup 网页抓取 + iText PDF 生成 + Knife4j 接口文档
